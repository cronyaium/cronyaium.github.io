怎样搭建一个博客（基于Linux&Hugo&Git&GitHub）

0x00. 前置知识

0x01. Linux

  • 有一个带有Linux系统或者Linux虚拟机（带图形界面）的电脑

  • 熟悉Linux的基本命令（比如说cd，mkdir，ls等）

  • 熟悉vim的简单使用

  • 参考资料：

    • runoob-Linux
    • 《鸟哥的Linux私房菜》（P.S 内容有一部分过时，选读即可）

0x02. Markdown

  • 这个比较简单，很容易上手，但也很有用处，写博客的时候基本上都是用Markdown，常见的***.md***后缀的文件就是用它写的

  • 不需要学到涉及到html的相关内容，当然能了解也很好

  • 参考资料：

    • runoob-Markdown

0x03. Git

  • 这个稍微复杂一些，可能需要一段时间 (也可能是我太菜了)

  • 建议顺便学习GitHub使用相关的一些内容

  • 参考资料：

    • runoob-Git
    • 《Pro Git》 (英文电子书，阅读过程中可能会有一些困难，建议查单词并在书中备注)

然后就米有辣！！学好了就可以开始啦！！

0x10. 本地环境搭建

0x11. 安装Hugo

以下所有操作都是在root用户（CentOS 8发行版）下进行

1. 下载Hugo包

wget https://github.com/gohugoio/hugo/releases/download/v0.88.1/hugo_0.88.1_Linux-64bit.tar.gz

可以根据最新版本修改网址（可以换成extended版本）

强烈建议科学上网

2. 安装

tar -zxf hugo_0.88.1_Linux-64bit.tar.gz

打包结束后，执行

sudo cp ./hugo /usr/local/bin

检查是否安装

hugo version

如果成功安装，会出现类似于下面的反馈

hugo v0.88.1-5BC54738 linux/amd64 BuildDate=2021-09-04T09:39:19Z VendorInfo=gohugoio

0x12. 新建一个网站

首先新建一个网站，执行

hugo new site [web]

其中[web]替换为你自己喜欢的目录

进入[web]目录

cd [web]/

然后会在目录下新建一批目录，包含一些配置文件

目录：archetypes  content  data  layouts  resources  static  themes

文件：config.toml

0x13. 配置theme

可以在这里查看你心喜的theme，这将成为你blog的主题

选择好之后，在这里找到对应的主题，然后下载下来

P.S 请提前安装好git

我使用的是tranquilpeak主题，首先进入themes目录

cd themes/

从远程仓库下载到本地

git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git tranquilpeak

不同的theme参照它的文档的指引进行

然后需要对[web]目录里的config.toml文件进行修改

先备份

cp config.toml config.backup

然后复制范例网站的config.toml文件覆盖[web]目录下的config.toml

cp ./themes/tranquilpeak/exampleSite/config.toml ./

选择覆盖，然后vim编辑config.toml文件，根据提示进行修改

0x14. 利用Hugo新建文章

首先cd回到[web]目录

然后新建文章

hugo new post/[name].md

其中[name]替换成你自己喜欢的名字

然后

cd content/post

vim [name].md

在vim中可以看到有配置的一些信息，在文末换行用Markdown写下你想写的内容，保存并退出

0x14. 启动hugo服务

cd回到[web]

hugo server --buildDrafts

成功后在FireFox访问

http://localhost:1313

就可以辣！！

0x15. 在本机上访问hugo搭建的网址

可以参考这里

0x20. 通过git将blog输送到GitHub

如果上面的都完成了，那么恭喜你已经可以在本地访问了！

但是我们如果希望要其他人也能通过网络访问到我们的blog，那么我们就需要通过git将blog输送到远程仓库GitHub

然后通过GitHub Page，我们就可以让其他人通过网络访问到我们的blog

0x21. 在GitHub中新建一个仓库

默认你已经会注册并且已经注册了一个GiIHub账户

仓库的名称起名为[你的用户名].github.io

0x22. 生成html文件

在blog目录下，执行

hugo

0x23. 将生成的html文件传输到GitHub

进入public目录

cd public/

然后初始化git仓库

git init

将工作树文件传输到暂存区

git add .

将暂存区文件传输到本地资源库

git commit -m "my first blog"

关联远程仓库

git remote add origin git@github.com:yourname/yourname.github.io.git

P.S 可能需要使用ssh密钥，参考度娘

将本地资源库文件传输到远程资源库

git push -u origin master

成功后就可以试着访问一下了
